takeshita kouhei

これまで

・1995年生まれ　宮崎県出身 高校卒業まで宮崎にて生活。
・高校では電機、機械、電子について幅広く学ぶ。授業にて、マイコンカーを制作し、C言語を使用し、プログラミングを行う。
・又、バドミントン部に所属し、副キャプテンになる。高校2年の新人戦にて団体で県2位になり、九州大会出場。
・高校卒業後、愛知県の電機メーカーの製造工場にて約7年、品質管理の業務に携わる。
・主な業務として、生産された製品の検査、機能試験、製造現場の不具合対応、試験設備の管理・修繕、検査要領書の作成・更新。
・退職後は、半年間、宮崎キャリア・デザインスクールにてデザイン、コーディング、マーケティング、写真撮影、動画編集について学ぶ。

きっかけ

・業務内にて業務効率化の為、VBAを使用し、エクセルのマクロを触るようになる。
→検査の際に使用するチェックシートの自動化を実施。
制作内容物件名入力し、検査品目を選択すると、チェックシートのエクセルに自動入力され、保存先フォルダを作成し、自動でPDF保存し、画面に表示させる。現在も使用されている。
その際に実際に入力したプログラム通りに動いたことが嬉しく、又楽しく、ITやプログラミングに興味を持つ。
・プライベートにてライブに行くようになり、ライブやフェスで配布されるフライヤーに興味を持ち、こういうものでなにかを発信できるようになりたいと思い、デザインについて学び、デザイン制作を仕事にしたいと考えるようになる。

コーダー、エンジニアへの興味

・職業訓練にてweb制作を学ぶ際にコーディングに興味を持ち、卒業制作では "Three,js" を使用し、2Dアニメーションを制作する。

興味のある技術

HTML　CSS　JavaScript　jQuery　SCSS　Three.js　WordPress　Illustrator　Photoshop　XD

強み

・前職では、検査業務に従事した為、人一倍品質やクオリティーに関しての意識は高いと感じています。
・現状を把握してから行動することを常に心掛けています。それは、効率良く物事を進める為に大事なことだと思っているからです。前職では、チームで作業をすることだけでなく、個人で仕事をする機会も多くありました。その両方に共通していたのは、まず現状を把握して、自分の能力に見合ったスケジュールを立てると言うことです。これらが出来ていないと仕事が間に合わない等の状況に陥ることもあったため、スケジュールを大切に業務を進めるように意識してきました。
・又現場の検査体制の再構築を行った際は、今の現場の人たちがわかりやすい動きやすい資料に作り直すことが求められました。まずは、実際の現場での作業を確認し、検査を行う作業者目線を盛り込むことを意識し、必ず現場の人に確認を取りながら納得のいくものを作成することを心がけていました。その結果、現場の人達から、見やすい・わかりやすくなったと高評価を得ることができました。これらは、初めに関係者で作成時のルール作成を行い、基準を決めたことでスムーズに業務を進めることができたと実感しています。そのため、現場を把握してから行動することを常に心がけるようになりました。

これから

・エンジニアの世界はトレンドや、技術の進化が早い為、常に新しい情報を取り入れていけるよう勉強や情報収集に力を入れていきます。
・又、自分の知識、技術を誰かの役にたてるよう発信する立場になりたいと思います。
・そして、関係する方々に寄り添い助けられるようなエンジニアになります。
